## Задача: RMQ — Минимумы на отрезках

Дан массив из `N` целых чисел. Необходимо обрабатывать `Q` запросов:
- Тип 1: `1 l r` — вывести минимум на отрезке `a[l..r]` (1-индексация, `l ≤ r`).
- Тип 2: `2 i x` — присвоить `a[i] = x`.

Чем эффективнее алгоритм (время на запрос/построение, память), тем больше баллов.

### Входной формат
- Первая строка: `N Q`
- Вторая строка: `N` целых чисел `a[i]`
- Далее `Q` строк запросов:
  - `1 l r` — минимум на отрезке
  - `2 i x` — точечное обновление

### Выходной формат
- Для каждого запроса типа `1 l r` выведите одно число — минимум на указанном отрезке.

### Пример
Ввод:
```text
5 6
5 3 7 3 2
1 2 5
2 3 1
1 1 3
1 3 3
2 5 10
1 4 5
```

Вывод:
```text
2
1
1
3
```

### Примечания
- Для статических запросов используйте `sparse table` (предобработка `O(N log N)`, ответ `O(1)`).
- Для запросов с обновлениями используйте дерево отрезков (`O(log N)` на запрос и обновление).
- Для оценки производительности можно поднять `N, Q` до `2e5` и при необходимости увеличить таймаут опцией раннера `--timeout`.
